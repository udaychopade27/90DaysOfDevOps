# Day 04 – Linux Practice: Processes and Services

## Objective
Practice Linux fundamentals using real commands to understand processes, services, and troubleshooting workflows.

---

## Process Checks

### 1. View All Running Processes

**Command:**
```bash
ps -ef
```

**Sample Output:**
```
UID        PID  PPID  C STIME TTY      TIME CMD
root         1     0  0 08:00 ?        00:00:02 /sbin/init
root       501     1  0 08:01 ?        00:00:00 sshd
ubuntu    1220  1100  0 08:20 pts/0    00:00:00 bash
ubuntu    1268  1220  0 08:25 pts/0    00:00:00 ps -ef
```

**Observation:** Shows all active processes and their Process IDs (PID).

---

### 2. Monitor Processes in Real Time

**Command:**
```bash
top
```

**Sample Output:**
```
Tasks: 125 total, 1 running, 124 sleeping
%Cpu(s): 5.2 us, 1.1 sy
MiB Mem : 3850 total
```

**Observation:** Used to monitor CPU usage, memory usage, and active processes.

## Service Checks

> Selected service: **Docker**

### 3. Check Docker Service Status

**Command:**
```bash
systemctl status docker
```

**Sample Output:**
```
● docker.service - Docker Application Container Engine
   Loaded: loaded
   Active: active (running)
```

**Observation:** Docker service is active and running.

---

### 4. List Running Services

**Command:**
```bash
systemctl list-units --type=service
```

**Sample Output:**
```
cron.service
docker.service
ssh.service
snapd.service
```

**Observation:** Displays all currently loaded system services.

---

## Log Checks

### 5. View Docker Service Logs

**Command:**
```bash
journalctl -u docker
```

**Sample Output:**
```
Docker daemon started successfully
API listening on /var/run/docker.sock
```

**Observation:** Used to inspect logs generated by the Docker service.

---

### 6. Monitor System Logs in Real Time

**Command:**
```bash
tail -n 50 /var/log/syslog
```

**Sample Output:**
```
May 21 systemd[1]: Started Docker Service
May 21 sshd[501]: Accepted password for ubuntu
```

**Observation:** Shows latest log entries — useful for troubleshooting.

---

## Mini Troubleshooting Scenario

**Scenario:** Verify whether Docker service is running properly.

### Step 1 – Check Service Status
```bash
systemctl status docker
```

### Step 2 – Inspect Service Logs
```bash
journalctl -u docker
```

### Step 3 – Check Running Process
```bash
pgrep docker
```

### Step 4 – Restart Service if Needed
```bash
sudo systemctl restart docker
```

### Step 5 – Verify Service Again
```bash
systemctl status docker
```
