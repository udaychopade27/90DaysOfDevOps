# 90 Days DevOps Learning Plan

## Current Level
I am currently working as a DevOps Engineer Intern and actively building hands-on experience with DevOps tools and cloud technologies. I want to strengthen my fundamentals and become industry-ready for a full-time DevOps Engineer role.

## My Understanding of DevOps & Cloud Engineering
For me, DevOps is a culture and practice that combines development, operations, automation, and collaboration to deliver software faster and more reliably.

Cloud Engineering is about building, deploying, monitoring, and managing scalable applications and infrastructure using cloud platforms and automation tools.

## Why I am Starting This Journey
I am starting this 90-day DevOps journey because I want to:

- Become a skilled and job-ready DevOps Engineer
- Gain practical hands-on experience with real-world tools
- Build confidence in problem-solving and troubleshooting

## My 90-Day Goals

1. Secure a full-time DevOps Engineer role with stronger technical skills and better opportunities

2. Build and deploy production-style DevOps projects using Docker, Jenkins, AWS, Terraform, and Kubernetes

3. Improve communication skills and become confident in explaining technical concepts

## 3 Core Skills I Want to Build

- Linux administration and troubleshooting
- CI/CD pipeline design and automation
- Kubernetes deployment and debugging

## Weekly Time Commitment

Weekdays:
- 3 hours daily
- Learning + hands-on practice

Weekends:
- 5–6 hours
- Revision + project work + live sessions

Approximate weekly learning time:
20–25 hours

## Consistency Plan

- Study every day, even if progress is small
- Practice by building projects instead of only watching tutorials
- Share progress publicly through #90DaysOfDevOps
- Revise concepts weekly and track progress

My goal is not only to complete 90 days, but to transform myself into a better DevOps Engineer through discipline and consistency.
